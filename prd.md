Product Requirement Document (PRD)Aplikasi Manajemen & Pembaca Disposisi Surat (SiDisposisi)1. Ringkasan Produk (Product Summary)Aplikasi ini bertujuan untuk mengotomatisasi proses input data dari dokumen surat disposisi berbentuk PDF ke dalam sistem database. Sistem akan membaca PDF secara otomatis, mengekstrak informasi penting, dan mendistribusikannya ke staf terkait. Staf penerima kemudian dapat melihat disposisi mereka dan mengunggah file dokumen balasan sebagai bukti tindak lanjut.2. Sasaran & Tujuan (Goals & Objectives)Mengurangi Input Manual: Mengubah proses pembacaan dan input surat disposisi dari manual menjadi otomatis menggunakan teknologi PDF Parsing/AI.Akuntabilitas: Memastikan setiap disposisi surat memiliki pelacakan status yang jelas (Pending\Completed).Efisiensi Dokumen: Menyediakan wadah terpusat untuk surat masuk dan dokumen balasan dari staf.3. Arsitektur & Stack Teknologi (Tech Stack)Frontend: Next.js (React Framework) + Tailwind CSS + Shadcn UI.Backend: Node.js (Express atau NestJS) untuk menghandle logika OCR/AI Parsing yang berat.Database: Supabase (PostgreSQL) untuk menyimpan data relasional.Authentication: Supabase Auth (Login Admin & Staf).File Storage: Supabase Storage untuk menyimpan file PDF Surat dan PDF Balasan.4. Peran Pengguna (User Roles)Admin / Sekretariat: Mengunggah PDF disposisi, memverifikasi hasil pembacaan sistem, dan mengelola data master.Staf (Penerima Disposisi): Melihat daftar disposisi yang diterima, membaca detail surat, dan mengunggah file balasan/tindak lanjut.5. Kebutuhan Fitur (Functional Requirements)Epik 1: Autentikasi & Manajemen Pengguna (Supabase Auth)Sistem harus mendukung login terpisah antara Admin dan Staf.Admin dapat mendaftarkan akun staf baru.Epik 2: Upload & Parsing PDF Disposisi (Sisi Admin)Upload PDF: Admin dapat mengunggah file PDF surat disposisi ke sistem (disimpan di Supabase Storage).Ekstraksi Otomatis: Node.js Backend akan membaca teks dari PDF tersebut dan mengekstrak komponen berikut:Judul/Perihal SuratNomor SuratTanggal Surat & Tanggal DisposisiNama/Jabatan Staf Penerima DisposisiCatatan/Instruksi DisposisiReview Form: Sebelum masuk database, hasil ekstraksi ditampilkan dalam bentuk form di Next.js agar Admin bisa mengedit jika ada kesalahan pembacaan, lalu menekan tombol "Simpan ke Database".Epik 3: Dashboard & Riwayat Disposisi (Sisi Staf)Staf dapat melihat daftar surat disposisi yang ditujukan kepada mereka.Terdapat filter status: Pending (Belum ditindaklanjuti) dan Completed (Sudah diunggah balasannya).Staf dapat mengunduh atau melihat langsung dokumen PDF disposisi asli.Epik 4: Tombol Upload & Tindak Lanjut (Sisi Staf)Pada detail disposisi berstatus Pending, sistem wajib menampilkan Tombol Upload File Balasan.Staf mengunggah file bukti tindak lanjut (PDF/Docx/Gambar).Setelah upload berhasil:File disimpan ke folder balasan-disposisi di Supabase Storage.Status disposisi berubah otomatis dari Pending menjadi Completed.Sistem mencatat waktu pengunggahan (timestamp).6. Skema Database (Supabase / PostgreSQL)Berikut adalah rancangan tabel yang perlu Anda buat di Supabase:Tabel profiles (Data Pengguna)id: uuid (Primary Key, link ke auth.users)full_name: textrole: text (enum: 'admin', 'staff')created_at: timestampTabel surat_masukid: uuid (Primary Key)nomor_surat: varcharjudul_surat: texttanggal_surat: datefile_url_asli: text (URL dari Supabase Storage)created_at: timestampTabel disposisiid: uuid (Primary Key)surat_id: uuid (Foreign Key ke surat_masuk.id)staff_id: uuid (Foreign Key ke profiles.id)catatan_instruksi: textstatus: text (default: 'pending', enum: 'pending', 'completed')created_at: timestampTabel dokumen_balasanid: uuid (Primary Key)disposisi_id: uuid (Foreign Key ke disposisi.id, Unique)file_url_balasan: text (URL dari Supabase Storage)uploaded_at: timestamp7. Alur Integrasi Sistem (Data Flow)Frontend (Next.js): Admin kirim file PDF \Kirim ke Backend (Node.js).Backend (Node.js): Memproses PDF menggunakan library parser/AI \Kembalikan data berupa JSON ke Frontend.Frontend (Next.js): Menampilkan JSON di form \Klik "Simpan" \Data dikirim langsung ke Supabase via Supabase Client SDK, sementara file PDF diunggah ke Supabase Storage.Staf Dashboard: Next.js melakukan real-time subscription atau fetch data dari Supabase berdasarkan staff_id pengguna yang sedang login.8. Kebutuhan Non-Fungsional (Non-Functional Requirements)Keamanan Data (RLS): Mengaktifkan Row Level Security (RLS) di Supabase agar Staf A tidak bisa melihat disposisi atau mengunggah file untuk Staf B.Performa Ekstraksi: Proses pembacaan PDF di backend Node.js tidak boleh memakan waktu lebih dari 5-7 detik per dokumen.Batasan Ukuran File: File PDF surat masuk maksimal 5MB, file balasan maksimal 10MB.
